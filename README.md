# Smart Logistics Dashboard – Analyse des retards logistiques

## 📌 Description du projet

Ce projet vise à analyser les retards logistiques à partir d’un jeu de données issu de Kaggle portant sur des opérations de transport intelligentes en 2024.

L’objectif est de construire un tableau de bord interactif avec Power BI afin d’identifier les principaux facteurs de retard, suivre les indicateurs clés de performance et proposer des pistes d’amélioration opérationnelle.

## 🎯 Objectifs

- Identifier les principales causes des retards logistiques.
- Suivre les KPIs clés liés à la performance logistique.
- Analyser l’impact du trafic, des actifs, du temps d’attente et des conditions environnementales.
- Visualiser l’évolution des retards au cours de l’année 2024.
- Aider à la prise de décision grâce à un dashboard interactif.

## 📊 KPIs analysés

- Nombre total de retards
- Taux de retard
- Temps d’attente moyen
- Utilisation moyenne des actifs
- Retards par actif
- Retards par condition de circulation
- Principales causes de retard
- Corrélations entre température, humidité, temps d’attente et retard

## 📈 Analyses réalisées

- Analyse descriptive des KPIs
- Analyse temporelle des retards
- Analyse des causes principales de retard
- Analyse par actif logistique
- Analyse de corrélation
- Visualisation interactive avec filtres

## 📌 Résultats principaux

- Le taux de retard global observé est d’environ 56,6 %.
- Le temps d’attente moyen est d’environ 35 minutes.
- Certaines causes comme la météo, le trafic et les pannes mécaniques apparaissent comme des facteurs importants.
- Certains actifs présentent un taux de retard plus élevé que d’autres.
- Les corrélations entre température, humidité et retard semblent faibles dans ce dataset.
